Data Loader Guide

Reference for the Protocol Data Loader — how to configure it, which scenarios to run, what data gets generated, and how lab panels and signal review gates are triggered.

⚡ Stress Test · 📦 Data Loader · 📖 Guide
Getting Started
All synthetic data writes to a separate Firestore collection (synthetic_logs) — completely isolated from your real logs. Click Clear Synthetic Data at any time to wipe it. Your real data is never touched.

Step-by-Step

  1. 1
    Open the Loader
    Navigate to 120-day-test.html. You can also reach it from the ⚡ Stress Test nav link at the top of any companion page.
  2. 2
    Sign In
    The Load button stays disabled until authenticated. If the status says "Checking login status…" for more than a few seconds, sign in through the main app first, then return.
  3. 3
    Configure Your Run
    Set days, start date, profile, data pattern, and the auto-set dates checkbox. See the Configuration Fields section below.
  4. 4
    Read the Preview Line
    Before clicking Load, the small text under the checkbox confirms your baseline date, NA start date, NA day count, and which signal review gates will be ACTIVE or past.
  5. 5
    Click "Load X Days of Data"
    A progress bar fills. The log shows every step — day generation, lab panel insertions, batch writes. Takes 15–60 seconds depending on day count.
  6. 6
    Go Back to the App
    Click Back to App. Charts, signal review, and PDF export are all live immediately.
  7. 7
    Clean Up When Done
    Click Clear Synthetic Data to wipe every document tagged _synthetic: true from Firestore. Real logs unaffected.
Configuration Fields
Days to generate
How many days of daily logs to create. Range: 7–365. Default 120. Change it and the button text, start date, and gate preview all update instantly. Tip: change this first, then adjust start date if needed.
Start date
The first day of synthetic history. Auto-sets to X days ago so today is always the final day of the generated range.
Profile
Which pilot profile to generate data for. Currently: Pilot (Patrick).
Data pattern
The shape of the data over time. Realistic, Volatile, or Flat — see Data Patterns below.
Auto-set dates
When checked, clicking Load writes baseline and NA start dates to localStorage. Leave checked unless dates are already set manually in the app.
Preview Line

What to Look For

Before you click Load, the preview under the checkbox reads something like:

Baseline start: 2025-11-24  |  NA start: 2025-12-08  |  NA day 106
Signal review gates: 3-month ✓ past, 6-month ✅ ACTIVE

This confirms exactly which gates will be open when you open the app — no surprises. If it shows a warning about needing more days to reach a gate, just increase the day count.

Data Patterns

Choose Your Scenario

Realistic Default

Best for: Full story arc, signal review narrative, PDF export quality.

  • Symptoms start high (~6.5/10) and trend down to ~4 over time
  • Energy starts low (~4/10) and climbs to ~7
  • Pain starts ~5.5 and drops to ~3.5
  • Sleep quality improves steadily throughout
  • Adherence grows from ~75% to ~95%
  • NMT level shifts from none/light → full by end
  • 15% flare day chance per day adds realistic noise
  • FAST markers (near-falls, spasms) decrease over time
Volatile Stress Test

Best for: Difficult/non-linear response, messy chart testing, signal review language for a patient not improving cleanly.

  • All metrics follow a ~7-day sine wave — good weeks, bad weeks
  • Slight overall improvement is buried in the noise
  • Symptom volatility scores are elevated
  • Flare days cluster at wave peaks
Flat Baseline / No Response

Best for: Non-responder scenarios, checking that charts don't falsely show improvement, testing the "no signal" case in signal review.

  • Everything holds steady — symptoms ~5, energy ~5.5, pain ~4, sleep ~6
  • Consistent adherence but no trend
  • No meaningful NMT progression
  • Flare days still occur randomly
Lab Panels

Milestone Blood Work

Lab panels are automatically embedded at milestone days within the range you generate. If you run fewer days, the final day gets the closest appropriate panel so the lab tab is never empty. VLCFA values trend gradually downward across all phases.

Auto-Set Dates

localStorage Keys

When the checkbox is checked (default), clicking Load writes these keys to localStorage. This is what unlocks signal review — without them, the app doesn't know when the protocol started and checkpoints won't fire. Uncheck only if you've already set dates manually in the app.

amn_baseline_start_date
Set to the start date you configured in the loader
amn_na_start_date
Set to start date + 14 days (nervonic acid protocol start)
